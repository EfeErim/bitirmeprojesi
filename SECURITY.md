# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please follow these steps:

### How to Report

1. **Do NOT open a public issue** - Security vulnerabilities should be reported privately to give us time to fix them before they are exploited.

2. **Email the security team**: Send your report to [security@aadsulora.com](mailto:security@aadsulora.com)

3. **Include the following information**:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Any proof-of-concept code or screenshots
   - Your name (optional) and affiliation

### What We Promise

- **Acknowledgement**: We will acknowledge receipt of your report within 48 hours.
- **Assessment**: We will investigate and respond with our assessment within 7 days.
- **Resolution**: We will work to resolve the issue as quickly as possible.
- **Credit**: You will be credited in the release notes unless you request anonymity.

### Responsible Disclosure Policy

We follow a responsible disclosure policy:

- **Initial Contact**: Report privately via email
- **Response Time**: We will respond within 7 days
- **Fix Timeline**: We aim to fix critical issues within 30 days
- **Public Disclosure**: We will coordinate public disclosure after the fix is released

### Scope

This security policy covers the following areas:

- **API Security**: Authentication, authorization, and data validation
- **Data Protection**: Personal data handling and encryption
- **Infrastructure**: Server configuration and deployment security
- **Dependencies**: Third-party library vulnerabilities

### Out of Scope

- Denial of service attacks
- Physical security of infrastructure
- Social engineering attacks
- Issues requiring physical access to devices

### Safe Harbor

We consider activities conducted in a manner consistent with this policy to constitute "authorized" conduct and will not initiate legal action against you. We will help you if legal action is initiated by a third party against you.

### Preferred Languages

We prefer reports in English, but will accept reports in Turkish if necessary.

### Bounties

Currently, we do not offer monetary bounties for security vulnerabilities. However, we deeply appreciate responsible disclosure and will provide public acknowledgment of your contribution.

## Security Best Practices

### For Users

- Keep your dependencies updated
- Use strong, unique passwords
- Enable two-factor authentication where available
- Monitor your account activity

### For Developers

- Follow secure coding practices
- Validate all user inputs
- Use parameterized queries to prevent SQL injection
- Implement proper authentication and authorization
- Keep dependencies updated
- Use HTTPS for all communications
